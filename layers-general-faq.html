What is Layers?

Layers is a extension of runtime resource overlay that was developed by sony to be used as a way for OEMs to provide a theming solution that they could use in android.

Why is root needed?

We use root to do this to provide a way for the android community can use regardless if they are on aosp or touchwiz or any other skin to provide versatility.

Why were changes needed on lollipop?

Our ability to theme resources were quite limited in lollipop since a lot of android linked back to the frameworks which we could not overlay with RRO so we had to make changes in AOSP source to provide a way to theme more. Thankfully Google saw this inability also and has made changes in M so there will need to be less that needs to be modified for theming and better support natively.

Why does the community say Layers or RRO?

When we started the project, we didn't have a name for this system and we just called it how we were using it and it stuck

So what is a Layer?

A layer is what we call our themes. A Layers theme is a collection of overlays

Where are layers stored to be loaded?

Since RRO was developed for OEM use, we use a location of system/vendor/overlay to place our theme apks.

Is Layers a theme engine?

No, there is no engine in place with Layers, it uses native android code to work wile theme engine was just naming by T-Mobile to market thier theming system back on gingerbread phones. Cyanogenmod has stuck with that naming and are way closer to it then what Layers is

So how does it work?

What a overlay does it that it looks for a string in the the resources and replaces it during loading into the system. For a way better and more in depth explanation see sony's blog post abot how RRO works http://developer.sonymobile.com/2014/04/22/sony-contributes-runtime-resource-overlay-framework-to-android-code-example/
